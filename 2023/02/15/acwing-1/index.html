排序

时间复杂度

指平均复杂度，一共logn层
快排和归并的时间复杂度都是O(nlogn)

快速排序

思想 —— 分治

  1. 确定分界点 q[l],q[(l + r) / 2],q[r]或随机
  2. 调整区间：分成两半，一半 <= x，另一半 >= x
  3. 递归处理左右两段

方法

  1. 暴力（需要两个空间）
  2. 更好的方法
    两个指针i和j，两指针向中间靠拢，指针i从左端找大于等于x的数，指针j从右端找小于等于x的数，然后swap二者，直至i和j相遇。

模板

要注意分界点的选值，如果是q[l]的话,把j换成i就会导致死循环

归并排序

思想——分治
与快排的区别：归并分界点是中间的下标值，快排是随机的一个数值

  1. 确定分界点：mid = (l + r) / 2
  2. 递归排序左右
  3. 归并：合二为一

方法

上下两个排序从初始点开始比较，更小的数放入新的排序，指针向后移，如果两数相等，选择上面的数放入新的排序，以此类推

以此类推。
和计算中乘法不同的是，并非A的每一项乘以b的每一项，而是A的每一项乘以b

前缀和

$S_i = a_1 + a_2 + … + a_i$, $S_0 = 0$
注意下标一定要从1开始,可以定义$S_0 = 0$,方便处理边界问题

  1. 如何求Si?
    使用一个for循环，递归 S[i] = S[i-1] + ai
  2. 作用
    能快速求出原数组中一段数的和，例如想求[l,r]这段数的和，如果用for循环的方式，时间复杂度为O(n),但如果用$Sr - S{l-1}$能够更快

推导至求一个二维矩阵的和，$S_{ij}$表示左上角所有元素的和,二维前缀和数组一般是从 1 开始而不是 0

模板

例：
题目描述：

假定有一个无限长的数轴，数轴上每个坐标上的数都是0。

现在，我们首先进行 n 次操作，每次操作将某一位置x上的数加c。

近下来，进行 m 次询问，每个询问包含两个整数l和r，你需要求出在区间[l, r]之间的所有数的和。

输入格式

第一行包含两个整数n和m。

接下来 n 行，每行包含两个整数x和c。

再接下里 m 行，每行包含两个整数l和r。

输出格式

共m行，每行输出一个询问中所求的区间内数字和。

数据范围

$−10^9≤x≤10^9$,
$1≤n,m≤10^5$,
$−10^9≤l≤r≤10^9$,
$−10000≤c≤10000$

如何实现去重unique

双指针算法，满足下面两个要求:

  1. 它是第一个
  2. a[i] 不等于 a[i - 1]

区间合并

思路

多个有交集的区间进行合并

  1. 按区间左端点排序
  2. 扫描整个区间，判断第二个区间与当前区间的关系
    • 在区间内部 不改变
    • 有交集 更新ed
    • 无交集 找到了一个区间，更新st和ed
